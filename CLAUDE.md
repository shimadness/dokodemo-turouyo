# どこでも釣ろうよ — 作業ルール

**まず [STATUS.md](STATUS.md) を読むこと。** 作業状況・設計判断・教訓・次タスクが全部そこにある。

## 必須ルール

- 手触りの数字は `src/game/main.js` 冒頭の `TUNE` に集約されている。バランス調整はまずここ
- SVGフィルタ（feGaussianBlur等）は使用禁止（モバイル描画負荷）。グローは疑似（同心円/二重ストローク）で
- ゲームループには必ず setInterval フォールバックを付ける（rAFが止まる環境がある）
- センサーは `orientationLive`（実データ到着）で判定する。`motionOn`（許可）と混同しない
- UIヒントは実状態で出し分ける（使えない操作を案内しない）
- 検証は `fishing.html?debug=1` の `window.__game` フックで（決定論シミュの型は STATUS.md 参照）
- main へ push すると本番へ自動デプロイされる（公開リポジトリ）。デプロイ後は
  `curl -s https://shimadness.github.io/dokodemo-turouyo/... | grep` で反映確認まで行う
- コミットメッセージは日本語で「何を・なぜ」
- **作業を終えたら STATUS.md の該当箇所（現在仕様・FB履歴・次タスク）を更新する**

## UX方針（プロジェクトオーナーの一貫した要望）

- 説明文を読まずに見て分かるUI
- 悪いアクションはゲージが減る等、行動の良し悪しが即フィードバックされる
- 実釣の理屈に寄せる（STATUS.md の調査結果参照）
- 子供（息子さん）も遊ぶ。タップだけでも全機能が成立すること（センサーは上乗せ）
